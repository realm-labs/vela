This page will describe how Vela uses type hints as metadata for analysis, dispatch, reflection, ABI checks, and selected runtime guards.

## Planned Coverage

- Type hints are not script-language generics.
- Hints on parameters, fields, globals, and return values.
- Guarded runtime behavior and diagnostic reporting.
- Interaction with hot reload compatibility.
- Interaction with host schemas and reflection metadata.
